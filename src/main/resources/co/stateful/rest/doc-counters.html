Counters are accessible through RESTful API. There are two operations availables on every counter: set and increment. Set returns nothing, while increment returns current value of the counter. In order to read without changing you just increment by zero. An increment with a negative value will decrement a counter.

For example, in order to set test counter to 123, you make a PUT request: /c/test/set?value=123. In case of success, HTTP response code will be 303.

In order to increment the same counter by 54, you make a GET request: /c/test/inc?value=123. Request header Accept should include text/plain. In case of success HTTP response code will be 200 and response body will contain counter value, in plain text format (integer number).

Maximum number of digits in a counter value is 32.

You can have up to 64 counters in one account.
